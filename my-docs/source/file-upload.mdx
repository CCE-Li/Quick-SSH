# 文件上传

Quick-SSH 提供两种文件上传方式：SSH 会话中的拖拽上传和独立的 `qssh-uploader` 工具。

## 拖拽上传

SSH 连接后，将本地文件或目录**拖入终端窗口**，Quick-SSH 会自动在新窗口中启动 SFTP 上传。

### 工作原理

当用户在终端中拖拽文件时，终端模拟器会将文件路径"粘贴"到输入流中。`drag_detect.rs` 模块负责检测这种拖拽行为：

1. **路径检测**：从输入流中提取文件路径
2. **路径验证**：检查所有 token 是否都是合法的绝对路径（防止误判普通命令）
3. **文件存在性验证**：确认路径在本地文件系统中确实存在
4. **启动上传器**：在新控制台窗口中启动 `qssh-uploader` 进行上传

### 检测策略

<Properties>
  <Property name="Windows 路径" type="X:\...">
    检测 `C:\Users\...` 格式的 Windows 绝对路径
  </Property>
  <Property name="Unix 路径" type="/home/...">
    检测 `/home/user/...` 格式的 Unix 绝对路径
  </Property>
  <Property name="引用路径" type="带空格的路径">
    支持双引号/单引号包裹的路径，如 `"C:\My Documents\file.txt"`
  </Property>
  <Property name="多文件" type="空格分隔">
    支持同时拖拽多个文件，所有路径必须都是合法的绝对路径
  </Property>
</Properties>

### 防误判机制

为防止将普通命令（如 `ls -la /home/user`）误判为拖拽操作，检测器要求**所有 token 都必须是绝对路径**才会触发上传。只要有一个 token 不是合法路径，就视为普通键盘输入。

## 独立上传工具

`qssh-uploader` 是一个独立的可执行文件，与主程序一起分发。

### 用法

```bash
qssh-uploader <host> <local_file> [remote_dir]
```

**参数：**

| 参数 | 说明 |
|------|------|
| `--host` | 服务器地址（必填） |
| `--user` | SSH 用户名（可选） |
| `--port` | SSH 端口（默认 22） |
| `--key` | 密钥文件路径（可选） |
| `--remote-dir` | 远程目录（默认当前目录 `.`） |
| 位置参数 | 要上传的本地文件路径 |

**示例：**

```bash
qssh-uploader mysrv ./myfile.zip /remote/path/

# 完整参数
qssh-uploader --host 192.168.1.100 --user root --port 22 --key ~/.ssh/id_rsa --remote-dir /tmp ./file.zip
```

### 功能介绍

- **并发上传**：最多 3 个文件同时上传
- **实时进度**：显示每个文件的进度条（百分比 + 传输量）
- **防闪退设计**：启动时延迟 300ms 等待控制台初始化；panic hook 循环等待 Enter，确保窗口不闪退
- **错误日志**：错误日志写入 `%TEMP%\qssh-uploader.log`
- **进度渲染**：使用 `#` 和 `-` 字符绘制进度条，兼容所有终端
- **CJK 支持**：正确处理中文、日文等双宽字符的显示宽度

### 上传输出示例

```
qssh-uploader - 文件上传工具

文件 1/3: project.zip     [########################] 100%  45.2MB/45.2MB
文件 2/3: config.tar.gz   [##################------]  72%  12.1MB/16.8MB
文件 3/3: README.md       [########################] 100%  2.3KB/2.3KB

进度: 2/3 个文件完成  |  耗时: 12s
```

## 内联上传（预留）

`qssh/src/ssh/upload.rs` 中预留了内联上传实现（当前被 `#[allow(dead_code)]` 标记）。该方案使用 `scp` 命令在同一终端内完成上传，不启动独立窗口。

设计要点：

- 每个文件调用一次 `scp`，上传完成后显示结果
- 上传期间 SSH 终端暂时冻结（stdin 转发暂停），但 stdout/stderr 仍正常显示
- 使用 `-o ControlMaster=no` 避免与现有 SSH 会话冲突

当前版本默认使用独立的 `qssh-uploader` 进程方案。内联上传模块保留以备将来可能的上传策略切换。
