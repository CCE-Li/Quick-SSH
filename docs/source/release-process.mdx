# 发布流程

Quick-SSH 使用 GitHub Actions 自动构建跨平台二进制并发布到 GitHub Release。

## 发布流程图

```
打 git tag v*
     ↓
GitHub Actions 自动构建
     ↓
生成各平台归档 + SHA256
     ↓
上传到 GitHub Release
     ↓
运行 update-packaging.ps1
     ↓
更新 packaging 目录下所有包管理器配置
     ↓
提交到各包管理器仓库
```

## 完整发布步骤

### 步骤 1：更新版本号

修改工作空间根目录的 [`Cargo.toml`](/Cargo.toml) 中的版本号：

```toml
[workspace.package]
version = "2.1.0"  # 更新此处
```

### 步骤 2：提交并打标签

```bash
git add .
git commit -m "chore: bump version to v2.1.0"
git tag v2.1.0
git push && git push --tags
```

### 步骤 3：GitHub Actions 自动构建

推送 `v*` 标签后，[`release.yml`](/.github/workflows/release.yml) 自动触发构建：

| 平台 | 归档格式 | 产物 |
|------|---------|------|
| Linux x86_64 | `.tar.gz` | `qssh-x86_64-linux.tar.gz` |
| macOS x86_64 | `.tar.gz` | `qssh-x86_64-macos.tar.gz` |
| macOS ARM64 | `.tar.gz` | `qssh-aarch64-macos.tar.gz` |
| Windows x86_64 | `.zip` | `qssh-x86_64-windows.zip` |

每个归档包含：

- `qssh` / `qssh.exe` — 主程序
- `qssh-uploader` / `qssh-uploader.exe` — 文件上传工具
- `LICENSE` — MIT 许可证
- `README.md` — 说明文档
- Shell 补全脚本（`qssh.bash` / `qssh.zsh` / `qssh.fish` / `qssh.ps1`）

### 步骤 4：更新包管理器配置

```powershell
.\scripts\update-packaging.ps1
```

此脚本自动更新所有包管理器配置文件中的版本号和 SHA256 校验和。

### 步骤 5：提交到各包管理器仓库

各包管理器的具体提交流程详见 [包管理器参考](/package-managers)。

## CI/CD 配置

### CI（持续集成）

[`ci.yml`](/.github/workflows/ci.yml) — 每次推送自动检查：

- 编译检查（`cargo check`）
- 运行测试（`cargo test`）
- Lint 检查（`cargo clippy`）

### Release（自动发布）

[`release.yml`](/.github/workflows/release.yml) — 打标签触发：

- 多平台交叉编译
- 生成归档并计算 SHA256
- 上传到 GitHub Release
- 生成 Shell 补全脚本

## 手动发布（备选）

如果 GitHub Actions 不可用，也可以手动构建：

```bash
# 1. 编译
cargo build --release

# 2. 创建归档
mkdir -p dist/Quick-SSH-v2.0.1-x86_64-pc-windows-msvc
cp target/release/qssh.exe dist/
cp target/release/qssh-uploader.exe dist/
cp LICENSE README.md dist/
cd dist && zip -r Quick-SSH-v2.0.1-x86_64-pc-windows-msvc.zip ./

# 3. 创建 GitHub Release
gh release create v2.0.1 dist/*.zip dist/*.tar.gz --generate-notes

# 4. 更新 packaging 配置
.\scripts\update-packaging.ps1
```

## 构建脚本

[`build.rs`](/qssh/build.rs) — 主程序的构建脚本：

```rust
fn main() {
    println!("cargo::rerun-if-changed=src/cli.rs");
}
```

补全脚本通过 `qssh completions <shell>` 子命令在运行时生成，构建时无需额外操作。Release 流程在 CI 中生成并打包补全文件。
